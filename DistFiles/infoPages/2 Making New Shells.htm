Making New Shells

The following instructions will guide you through making a basic Bloom shell which contains source text in a single language. At this time, Bloom doesn't have anything special for helping you make or distribute shells... that will come in the future.

Overview

For now, to make a basic shell, you just use Bloom as though you were making a new vernacular book. But instead of making the book in a vernacular library project, you make a library for the source language (e.g. "English").  Then create a new book, using a paper/layout template like "A5 Portrait".

Note: although Bloom supports multi-lingual shells, the Bloom user interface does not yet let you switch other languages (e.g., English and Tok Pisin).  The way to do that for now is to quit bloom, open up the "bloomLibary" document in notepad, and change the iso639 code and languageName to that of another language.  Save that, run Bloom again, and now you'll be adding a different national language to the same shells.

Using the Shell

In the future, we hope to have a nifty system of submitting and distributing shells, so that you can publish your collections and Bloom users can subscribe to them. For now, we have nothing so fancy as that.

OK, so you have a one or more shells in a collection folder, perhaps in your "My Documents/Bloom/my super collection" folder. Now you want to test them out or give them to someone else to use. What you need to do is to move that collection folder to where Bloom looks for shell collections. On Vista and Windows 7, this is found at "C:\Program Data\SIL\Bloom\Collections". On Windows XP, it is in "C:\Documents and Settings\All Users\Application Data\SIL\Bloom\Collections". You can either copy your shell collection into this folder, or create a shortcut there which points to your collection.

On your own computer, just use the shortcut method. That way, you can open a vernacular library and test out your new shell, without having to move it around the computer.
