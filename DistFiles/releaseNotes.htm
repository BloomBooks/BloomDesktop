(these notes were last updated for version 0.1.57.  Because we release so frequently (even daily), we may at times fall behind updating it.)

Known Problems

After extracting audio from a video file, the audio file does not appear in the file list. Work around: switch to another session, then back. [SP-99]

Changing the spelling of a custom-field name does not propagate the change to other files using the same custom-field. [SP-100]

In the “New Files From Device” dialog, if you click "Create Sessions" while the preview player is still running, it “locks” the file and SayMore cannot then copy it into a session. [SP-98]

Known Limitations

The Export to CSS command is only partially implemented.

There is no way  yet customize the file naming conventions.

There is no good way yet to track who all the contributors were to various session components (e.g., who did the transcribing). Until this is done, its possible to make do by using with custom fields.

While type-ahead works  on many fields, it does not yet work for entering multiple participant names.

You must currently use right-clicking on files to do things with them; this will soon be possible from the main-menu bar.

When using the “Add” button, you can only choose one file at a time.
